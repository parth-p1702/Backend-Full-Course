# Node.js REPL Lifecycle

## 1. Introduction

-   **REPL** stands for **Read--Eval--Print--Loop**.\
-   It is an **interactive shell** that comes with Node.js.\
-   Developers use it to quickly **test JavaScript/Node.js code
    snippets** without creating a file.\
-   Similar to Python's interactive shell or the browser console.

ðŸ‘‰ To start REPL:

``` bash
node
```

This opens the Node.js interactive environment.

------------------------------------------------------------------------

## 2. How REPL Works (Lifecycle)

The REPL follows **4 steps** in a loop:

1.  **Read** â†’ Takes user input (JavaScript code).\
2.  **Eval (Evaluate)** â†’ Interprets and executes the input using the V8
    engine.\
3.  **Print** â†’ Displays the output/result of the execution.\
4.  **Loop** â†’ Waits for the next input and repeats the cycle until user
    exits.

ðŸ‘‰ Example:

``` bash
> 2 + 3
5
> console.log("Hello")
Hello
undefined
```

Here:\
- **Read:** `2 + 3`\
- **Eval:** Computes it â†’ `5`\
- **Print:** Displays `5`\
- **Loop:** Waits for more input

------------------------------------------------------------------------

## 3. REPL in CLI (Command Line Interface)

-   Run REPL in CLI by typing:

``` bash
node
```

-   Some useful commands in REPL:
    -   `.help` â†’ Show all commands\
    -   `.exit` â†’ Exit REPL (or press `Ctrl + C` twice)\
    -   `.editor` â†’ Enter multi-line editor mode\
    -   `.save filename.js` â†’ Save session to file\
    -   `.load filename.js` â†’ Load file into REPL

ðŸ‘‰ Example:

``` bash
> let a = 10
undefined
> let b = 20
undefined
> a + b
30
```
